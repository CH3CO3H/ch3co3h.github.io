# 用算法思维解决九连环

最近到手了一个九连环，花了些时间自己解开了。在解决的过程中我体会到了具备算法思维对解决问题的重要性，于是写在这里，总结一下。

1. 拿到东西后，先观察结构，确定当前状态和最终目的；
2. 确定可以执行的操作；
3. 从初始状态开始，对每一步进行详细的记录，这样有助于分析规律寻找解法，也可以在想还原时按步骤还原；
4. 找到每个阶段之间的规律。

九连环有一个横杆，9 个竖杆，9 个环。最初状态是 9 个环都套在横杆上，最终状态是 9 个环与横杆分离。每个环有三种状态：套在横杆上，在横杆上方，在横杆下方。最终状态是要让 9 个环都处于在横杆下方的状态（也就是完全脱离）。我用右手持杆，左手解环，将环从左到右标记为 1~9。

在最开始的尝试中，我断定了整个解的过程就是递归操作，环要从 1 至 9 依次解下。虽然经过随意操作有可能解到第 5、6 个环，但尚未找到规律时，还是没法继续。其实最重要的就是前 3 个环的拆解，在开始解的时候一定要仔细记录，而且要一步步循序渐进。我的原始记录如下。

➀ 1ld

➁ 12ld

➂ ➁ 12u 2i 1d 23l 3d 2i 1ui 12ld {1~3d}

l: 从套在横杆的状态中解下；

u: 将环移到横杆上方；

i: 将环套在横杆上；

d: 将环移至横杆下方。

{ } 表示当前状态，{ 1~3D } 的含义是 1~3 环已解下，其余环在杆上。

其实在操作的过程中，我首先注意到整个状态从 { 12D } 经几步操作后直接转到了 { 1~4D }，这时我猜想，要解除环 k，须先装上环 k-1，将环 k 解下后置横杆下\(d)，再解下环 k-1。于是如何从 { 12D } 转到 { 1~3D } 就十分关键。按照猜想，我先装上 2 取下 1，这时状态为 { 1D }，再解除 23，这时状态为 { 1D2I3D }，然后解除 2，即转到 { 1~3D }，猜想得证。这时可以发现上面的 ➁ 可以写成这样。

➁ ➀ 1ui 12lu 2d 1d

按照这个步骤，将后面 4~9 全部解下，九连环就这样被我破解了。

将这个思路用算法表示出来，就是这样。

```
d (k)         // { 1~(k-1)D } -> { 1~kD }
	i (k-1)   // { 1~(k-2)D }
	ld (k)    // { 1~(k-2)D(k-1)IkD }
	d (k-1)   // { 1~kD }
```

函数 d\(k) 表示将状态 { 1~(k-1)D } 转为状态 { 1~kD }，也就是解下环 k 的操作。ld\(k) 是单步操作，那么还需要一个 i\(k) 函数，它表示安装上环 k。同样，想要安装环 k，则先要安装 k-1，装上环 k 再解下 k-1，即是这样。

```
i (k)
	i (k-1)
	ui (k)
	d (k-1)
```

ui\(k) 也是单步操作，表示将环 k 从下方移至上方再安装到横杆上。

解法是搞出来了，可是它是否有什么问题呢？我们把 d\(k+1) 展开一下。

```
i (k)
ld (k+1)
d (k)
```

接在 d\(k) 的后面。

```
d (k)
i (k)
ld (k+1)
d (k)
```

d\(k) 和 i\(k) 是互逆操作，它们两个可以直接抵消掉，再前一个操作是 d\(k-1)，连起来是这样。

```
d (k-1)   // { 1~(k-1)D }
ld (k+1)
d (k)     // { 1~(k+1)D }
```

也就是说解环是每两个环一个周期操作的，安装的函数就不用变化了。函数 d\(k) 本来表示由状态  { 1~(k-1)D } 转为状态 { 1~kD }，现在它可以表示由状态  { 1~(k-2)D } 转为状态 { 1~kD } 。

```
d (k)
	ld (k)
	i (k-2)
	d (k-1)
```
