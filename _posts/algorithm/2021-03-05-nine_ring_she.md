# 我和妹子解九连环思路的不同

先对上一篇文章勘个误。上一篇文章最后修改了 d\(n)，却说 i\(n) 不用改动了，这是不对的。因为 函数 i 调用了 d，d的含义改变了 i 当然要跟随变化。具体的改动，在 write.as 和 github.io 博客上都已完成。而发现这个错误的过程，便是这篇文章的内容。

最开始我是给妹子讲解如何把解九连环的步骤用算法的方式表示出来，首先要考虑大步骤。我自己的想法是从 1 到 n 的顺序，每一步解两个环。如果总共有奇数个环，则从 1 开始解，偶数则从 0 开始。当然九连环属于奇数，即这种方式：

![](https://i.loli.net/2021/03/05/vFT9xBIJjKwaG4L.jpg)

```
solve (n)
	start = n&1
	for i=start;i<n+1;i+=2
		d (i)
```

而妹子考虑的大步骤是从 9 到 1 的顺序，是这样的：

![nine-ring-she.jpg](https://i.loli.net/2021/03/05/nLEjCGdo94wiTKl.jpg)

```
solve (n)
	solve_r(n)
	solve(n-1)
```

`solve_r` 函数表示解下尚未解除的编号最大（也就是最里面的环）。这个函数是这样的：

```
solve_r (n)
	solve (n-2)
	down (n)
	install (n-2)
```

down 是放下单个环的单步操作。整个函数的含义是，要想解除最大的未解除环 n，需要先解除 1 ~ n-2，放下环 n，再安装上 1 ~ n-2。install\(n) 函数表示从 1 到 n 全部装上，具体内容是这样的：

```
install (n)
	install (n-1)
	solve (n-2)
	up (n)
	install (n-2)
```

up 和 down 相对应，指安装单个环的单步操作。前两步表示装上环 n-1，然后装上环 n，最后装上环 1 ~ n-2

按照这个算法写好代码，运行后计算出了解九连环的操作数。令我比较惊讶的是得出的数字与网上查到的解九连环操作步数相等；而我先前自己的想法计算出的数要小一些。当时我还并没觉得自己错了，因为思路并没问题。后来仔细一检查，才发现了错误。

我想，这两种解法或许存在着什么关联。我的解法分 n/2+1 个大步骤，妹子的解法分 n 个。我的前 n/2 步被完整包含在妹子解法的第一步，妹子解法的后 n-1 步被完整包含在我的解法的最后一步。从通用性上讲，妹子的解法更好，因为我的解法还要根据 n 的奇偶性做不同处理。
