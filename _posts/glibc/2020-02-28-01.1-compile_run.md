# 编译和运行

## 基本操作

打开任意文本编辑器，输入以下内容：

```c
/* inertia.c
	example for compiling and running. */

#include <stdio.h>

double cylinder(double m, double r);

int main(void) {
	double mass=5.0;
	double radius=1.2;
	double inertia=cylinder(mass, radius);
	printf("%lf\n", inertia);
	return 0;
}

double cylinder(double m, double r) {
	return m*r*r;
}
```

保存为 inertia.c。

使用以下命令编源译文件为可执行文件，并运行。

```shell
gcc inertia.c -o inertia && ./inertia
```

可以看到终端输出的结果。

```
7.200000
```

代码中的 /\* 到 \*/ 之间的内容叫做注释，我们可以在代码的任何地方添加注释，做适当的解释说明，编译器会把注释忽略掉。除了这种多行注释格式外，还有一种单行注释。

```c
// one line exn
```

从 // 到行尾都会被编译器忽略。

## 多余的知识

在 Windows 10 打开某个 Linux 子系统或者在某 Linux 系统里打开 Terminal 后看到的大黑界面叫做终端。我们在终端里输入指令，告诉电脑要去做什么，然后电脑会去执行相应的操作，同时在终端返回一些信息或者不返回。在学习编程前，我假设你已经对「文件」、「文件夹」和「文件路径」这些概念有了最基本的认识。

终端有一个「当前文件夹」的概念，意思是你正处在哪个文件夹下进行操作。上面那条命令，前半部分 `gcc inertia.c -o inertia` 表示用 gcc 编译并链接源代码文件，生成名为 inertia 的可执行文件。后半部分 `./inertia` 表示执行当前文件夹下的 inertia 可执行文件。用 `&&` 把它们连接起来写到一行，意思是上一条命令执行成功后才执行下一条。
